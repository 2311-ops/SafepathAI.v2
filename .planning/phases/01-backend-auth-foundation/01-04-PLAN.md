<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> /auth/forgot-password | Attacker-controlled email input; potential enumeration + email-spam vector |
| API -> Resend | Reset link leaves the system boundary via a third-party email provider |
| Reset link -> /auth/reset-password | Bearer-style token in a URL returns as untrusted input |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-01 | Information Disclosure | Reset-token leakage via DB, logs, or referrer | high | mitigate | Store only `IPasswordHasher.Hash(token)`, never the raw token or full URL; never log the token; single-use + 24h expiry (Task 1) |
| T-04-02 | Information Disclosure | Account enumeration via forgot-password | medium | mitigate | Identical 200 response and no email for unknown addresses (Task 1, asserted by tests) |
| T-04-03 | Denial of Service / Spoofing | Email-spam / reset-token brute force | medium | mitigate | Rate-limit `/auth/forgot-password` and `/auth/reset-password`; tokens are 32-byte CSPRNG so guessing is infeasible (Task 2) |
| T-04-04 | Information Disclosure | Resend API key in source | high | mitigate | Key lives in user-secrets/env, read from `Resend:ApiKey` config, never committed (checkpoint + Task 2) |
| T-04-SC | Tampering | `Resend` NuGet package supply chain | medium | mitigate | Resend is the official first-party SDK, VERIFIED + Approved in RESEARCH Package Legitimacy Audit; version pinned 0.5.1 |
</threat_model>
