<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Guardian/invitee app -> /families & /invites | Client actions on family data; server re-authorizes every one (plan 05) |
| Invite code/link -> device share sheet | The invite secret leaves the app via share_plus |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-07-01 | Elevation of Privilege | Client trusting a role/familyId it holds locally to show Guardian actions | high | mitigate | UI gating is convenience only; the server re-verifies membership+role on every call (plan 05 FamilyAuthorizationService) — the app treats a 403 as authoritative |
| T-07-02 | Information Disclosure | Invite link/QR over-shared | medium | mitigate | Server enforces 24h expiry + single-use + rate-limited redeem (plan 05); the QR encodes the longer opaque link token, and codes expire |
| T-07-03 | Tampering | Accidental destructive removal | low | mitigate | "Remove from circle" is confirmation-gated (dialog) and never fires on a single tap (UI-SPEC) |
| T-07-04 | Information Disclosure | Tokens attached to family calls | high | mitigate | family_api uses the shared dio auth interceptor (plan 02) so tokens come from secure storage, never hardcoded/logged |
</threat_model>
