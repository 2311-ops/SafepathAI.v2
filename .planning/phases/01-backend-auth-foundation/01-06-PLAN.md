<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User -> /auth/forgot-password | Email input; enumeration surface handled server-side + client copy |
| Deep link -> /reset-password | token+email arrive from an external email link as untrusted params |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-06-01 | Information Disclosure | Confirmation copy revealing account existence | medium | mitigate | Enumeration-safe confirmation text ("If an account exists...") regardless of input (Tasks 1-2) |
| T-06-02 | Information Disclosure | Reset token in logs / navigation history | medium | mitigate | Token used only to POST /auth/reset-password; not logged; screen does not echo the token; server enforces single-use + expiry (plan 04) |
| T-06-03 | Tampering | Deep link handling of attacker-supplied params | low | accept | token+email are validated server-side; the client only forwards them — residual risk accepted |
</threat_model>
