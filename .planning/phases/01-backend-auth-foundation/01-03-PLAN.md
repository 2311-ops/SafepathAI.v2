<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input -> app -> API | Email/password/role entered on-device, sent to /auth/* |
| App -> device secure storage | Returned tokens persisted locally |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-01 | Information Disclosure | Login error copy revealing which field is wrong (aids credential stuffing) | high | mitigate | Single enumeration-safe message "Incorrect email or password. Try again." for both unknown-email and wrong-password (Tasks 1-2, matches server behavior in plan 01) |
| T-03-02 | Information Disclosure | Tokens persisted insecurely | high | mitigate | Tokens saved only via `TokenStorage` (flutter_secure_storage) from plan 02; controller never writes them elsewhere |
| T-03-03 | Spoofing | Unauthenticated access to /home | medium | mitigate | go_router redirect gates `/home` on token/auth presence |
| T-03-04 | Information Disclosure | Password field shoulder-surf / logging | low | accept | Password field obscured; no logging of credentials — residual risk accepted for this phase |
</threat_model>
