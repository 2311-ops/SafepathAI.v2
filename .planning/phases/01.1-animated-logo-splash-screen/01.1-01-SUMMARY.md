# Phase 01.1 Plan 01: Animated Logo Splash Screen (Widget + Router Wiring) Summary

**Additive `/splash` cold-launch screen — SafePathLogo+wordmark lockup fading/scaling/rising in on a single 1400ms `AnimationController` (220ms opacity-only under reduced motion), gated into `routerProvider`'s redirect via a new `splashAnimationCompleteProvider` and an additive `_AuthRefreshListenable` listener.**

- **Duration:** ~15 min
- **Started:** 2026-07-10T22:20:00Z (approx.)
- **Completed:** 2026-07-10T22:39:42Z
- **Tasks:** 2

## Accomplishments
- Created `splashAnimationCompleteProvider` and `SplashScreen`, a `ConsumerStatefulWidget` reusing `WelcomeScreen`'s exact hero gradient + `SafePathLogo(size: 96, tile: false)` + `AppTypography.display` wordmark lockup, animated (fade 0→1 / scale 0.92→1.0 / rise 8px→0px) over `Interval(0.0, 0.55, Curves.easeOutQuart)` on a single 1400ms controller, with a deterministic opacity-only 220ms reduced-motion path and an exactly-once completion flip guarded by `_flipped`/`mounted`.
- Wired `/splash` into `routerProvider`: new additive `GoRoute`, `initialLocation` changed from `/` to `/splash`, `_AuthRefreshListenable` now also listens to `splashAnimationCompleteProvider`, and a `/splash` redirect gate (placed before the existing recovery rule) holds on `/splash` until the provider flips true, then routes to `/reset-password` / `/home` / `/` per the existing auth state — with zero changes to any other route, guard set, or redirect rule.

## Decisions Made
- Swapped the plan's literal `StateProvider<bool>` for a `Notifier<bool>` + `NotifierProvider` with an explicit `set()` method, because this project's `flutter_riverpod: ^3.3.2` has no legacy `StateProvider` API (`flutter analyze` failed with `undefined_function` on first attempt). Matched the existing `PendingInviteNotifier`/`ResetLinkExpiredNotifier` convention in `mobile/lib/core/deep_link/deep_link_service.dart` — same `Notifier` base class, same `set(...)` method name, same `ref.read(provider.notifier).set(...)` call-site pattern. No architectural change: still a single boolean piece of Riverpod state, same call sites the plan specified (`ref.read(...).notifier...`, `ref.listen<bool>(...)`, `ref.read(splashAnimationCompleteProvider)`).
