<objective>
Add an additive animated logo splash screen shown exactly once on cold launch, then hand off to the existing router destination (Home if authenticated, Welcome if not, reset-password on recovery) with zero changes to existing auth, session, or routing semantics.

Purpose: Give SafePath AI a calm, on-brand cold-launch moment that reuses the WelcomeScreen hero lockup, without touching the SOS-critical app shell or any existing screen. The completion-gate architecture is future-proofed for async init even though init is synchronous today.
Output: A `SplashScreen` widget, a `splashAnimationCompleteProvider`, and an additive `/splash` route + redirect gate wired into the existing `routerProvider`.
</objective>
