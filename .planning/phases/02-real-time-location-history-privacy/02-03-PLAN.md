## Phase Goal

**As a** family member, **I want to** see my family's live and past location on a shared map and control exactly what I share and with whom, **so that** we can keep each other safe without giving up our privacy.

<objective>
Deliver the server-authoritative sharing matrix (D-07) and wire it into the location pipeline so privacy is enforced in the backend â€” not merely as a client-side UI gate. Introduce `SharingPreference` as a NEW entity alongside FAM-04's `PermissionLevel` (RESEARCH Pattern 7 â€” the existing per-member enum cannot express "share history with Guardian A but not Guardian B"), a per-data-type Ă— per-recipient toggle, temporary time-boxed sharing that auto-stops (D-08), and a double-gate (`membership AND SharingPreference`) applied to every broadcast and read.

Purpose: This is the phase's primary IDOR/privacy-enforcement surface. Per the threat model, disabling a toggle must actually stop data flow at the server, and the check must run at broadcast time (not only at group-join) so an in-flight toggle change takes effect. Confirms PRIV-01 transport (WSS/HTTPS) is enforced.
Output: `SharingPreference` schema + migration, `ISharingAuthorizationService` gate, toggle/matrix commands, a sweep `BackgroundService`, `PrivacyController`, and the gate retrofitted into P02's `ReportLocationCommandHandler` + `GetLiveLocationsQueryHandler`.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client â†’ PATCH sharing-preferences | Client-supplied recipient/dataType/enabled/expiry; owner must be forced to caller |
| SharingPreference state â†’ broadcast/read | Privacy decision that must be enforced server-side, not client-side |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-02 | Information Disclosure | Privacy toggle enforced only client-side | high | mitigate | Gate lives in `ISharingAuthorizationService`, called by `ReportLocationCommandHandler` (broadcast) and `GetLiveLocationsQueryHandler` (read); the client toggle is purely a UI reflection of server state |
| T-02-01 | Tampering | Setting sharing prefs for another user's data | high | mitigate | OwnerUserId forced to `CallerUserId` in the command handler; no owner id accepted from the client body |
| T-02-05 | Information Disclosure | Race: disable toggle while a broadcast is in flight | medium | mitigate | FilterRecipients evaluated at broadcast time inside the handler, not cached at group-join |
| T-02-09 | Tampering | Expired temporary share still leaking data | medium | mitigate | Gate excludes `ExpiresAtUtc <= now` at read/broadcast time AND the sweep flips IsEnabled=false (defense in depth) |
| T-02-01b | Information Disclosure | GetSharingMatrix exposing another user's matrix | high | mitigate | Query scoped to `CallerUserId` after `RequireMembership`; only the caller's own rows returned |
</threat_model>
