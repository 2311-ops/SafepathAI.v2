<objective>
Complete the family-map identity slice: render every view-eligible member as a circular avatar (or default initial-letter avatar, D-18) with an always-visible name label, online/offline dot, and live position (PROFILE-06), and make name/picture changes appear in real time by consuming the `ProfileUpdated` SignalR event (D-17). Family-Circle-only visibility (PROFILE-07) is inherited from the backend `GetLiveLocationsQuery` double-gate — the client renders exactly what it is sent and introduces no new visibility logic.

Purpose: This closes the loop opened in 02-14 (backend `ProfileImageUrl` snapshot + `ProfileUpdated` broadcast) and 02-15 (avatar widget + cached_network_image). It extends `MemberMapPin` and `_LiveMemberMarker` IN PLACE (D-18) rather than building new marker widgets, and structures markers so they remain `flutter_map_marker_cluster`-compatible without adding that dependency (D-19).

Output: extended `LiveLocation` model, a `ProfileUpdated` hub stream + controller merge, and avatar-bearing, name-labelled map markers that update live.

MVP slice framing (ROADMAP Phase 2 Success Criterion #6): "...every visible family member appears on the live map as a custom marker showing their avatar (or default), name, online/offline status, and current location, updating in real time." This plan is that sentence, made real.
</objective>
