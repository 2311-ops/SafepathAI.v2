## Phase Goal

**As a** family member, **I want to** see my family's live and past location on a shared map and control exactly what I share and with whom, **so that** we can keep each other safe without giving up our privacy.

<objective>
Close the 02-VERIFICATION.md PRIV-03 UI partial gap. Backend temporary sharing and expiry already work, but the mobile Privacy Center currently starts temporary sharing against an implicit recipient and maps Custom to a fixed preset. This plan makes the recipient choice explicit in the UI and makes Custom collect an actual duration.

Purpose: Temporary sharing is a privacy control. If the user cannot choose the recipient and duration, the UI does not satisfy D-08 or PRIV-03 even if the backend expiry is correct.
Output: per-recipient temporary share controls, a custom duration dialog/input, and widget tests for non-first-recipient and custom-duration flows.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| user UI choice -> privacy controller | Recipient and duration are user-controlled privacy inputs that must not be silently substituted by the UI. |
| privacy controller -> backend PATCH | Server remains authoritative for enforcing the temporary sharing row and expiry; UI only sends the selected recipient and duration. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-02-11-01 | Information Disclosure | Temporary sharing recipient selection | high | mitigate | Controls are bound to explicit recipient rows and tests assert non-first recipient IDs are sent correctly. |
| T-02-11-02 | Tampering | Custom duration input | medium | mitigate | Dialog validates empty, non-numeric, zero, and negative values before constructing the Duration. |
| T-02-11-03 | Repudiation | User cannot tell who is temporarily receiving live location | medium | mitigate | Recipient-scoped controls and active-share remaining-time context are rendered under the relevant recipient. |
| T-02-11-04 | Denial of Service | Excessively large custom duration causing unintended long share | low | mitigate | Custom input validation caps duration at 7 days while preserving preset behavior. |
</threat_model>
