# Phase 2 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Scope:** Live/shared family map + presence (LOC-01..02), stale-location + accuracy indicator (LOC-03), battery-transparency screen (LOC-04), permission-priming screen (LOC-05), history timeline/route/stats (HIST-01..03), low-battery alert surfacing (NOTIF-01), and the full Privacy Center (PRIV-01..05). Maps to design-system feature sets **02-1 "Home / Live Map"** (map/presence only — the interactive press-and-hold SOS button itself is Phase 3/DESIGN-02, out of scope here), **05-3 "Privacy center"**, **06-1 "History timeline"**, **06-2 "Route + stats"**, and the Battery filter chip of **05-1 "Notifications center"**. Explicitly OUT of scope: the SOS trigger/arm/resolution flow (02-2..02-6), Walk-Me-Home (03), AI Insights/Safety dashboard (04), Visibility ledger 05-2 (Phase 6 LEDGER), Geofencing 06-3/06-4 (Phase 4), Health (07), Settings (08).

No RESEARCH.md exists yet for this phase (research not yet run) — stack/library UI-relevant questions (map SDK usage, component approach) are answered directly below from `.claude/CLAUDE.md`'s fixed stack and the existing Flutter codebase; nothing here is blocked on RESEARCH.md.

## Scope Resolutions (judgment calls made here, flagged for planner/executor)

### 1. Bottom-nav shell — Phase 2 is the first phase that needs it

Phase 1 deliberately shipped **no** bottom-nav shell (see `01-UI-SPEC.md` Scope Resolutions #2) because none of its 7 screens needed it, and instead shipped a deliberately-undesigned authenticated landing stub. Phase 2's Home/Live Map screen (02-1) is the design system's actual "Home" screen and **does** carry the persistent bottom nav (`Map / Activity / SOS / Insights / Privacy`, SOS raised center).

**Resolution:** Build the full 5-tab bottom-nav shell now, replacing Phase 1's plain stub, since Home/Live Map genuinely needs it and re-doing the app-shell chrome per-phase would be wasted work. Tab routing this phase:
- **Map** → Live Map screen (real, this phase — LOC-01/02).
- **Activity** → History Timeline + Route/Stats (real, this phase — HIST-01..03).
- **SOS** → render the raised 64px red circle **exactly to the visual spec** (per `SYSTEM_DESIGN (1).md` "The SOS Button" section: red `#DE3B40`, 4px white border, `SOS` label 800/15px white, raised -40px, drop shadow `0 10px 22px rgba(222,59,64,.45)`) but **non-functional this phase** — no press-and-hold arming, no `AnimationController`. A single tap may show a plain, non-alarming "Coming soon" toast/snackbar. Do NOT wire any real SOS logic, animation, or network call here — that is Phase 3's DESIGN-02 to build. Visually complete, functionally inert.
- **Insights** → not built this phase (Phase 5 AI dashboard). Render a simple placeholder ("Insights are coming soon") using default, undesigned Material styling — same "deliberately obvious placeholder" convention Phase 1 used for its landing stub, so nobody mistakes it for a finished screen.
- **Privacy** → Privacy Center (real, this phase — PRIV-01..05).

### 2. "Delete my data" color — mockup shows red, project directive forbids it

`SYSTEM_DESIGN (1).md`'s Privacy Center (05-3) literally annotates "Your data (Export / **Delete=red**)". However, this phase's explicit brief (`.claude/CLAUDE.md` / orchestrator instructions) states SOS red must **never** be reused for this phase's privacy UI, even though a data-delete action is "alert-like." Phase 1's single flagged exception ("Remove from circle" using `sosRedDeep`) was scoped narrowly to that one Phase-1 label only and does not carry forward automatically.

**Resolution:** "Delete my data" (PRIV-04) renders in **Ink** (`#15302E`), 700 weight, **not** any red-family color — on a plain/neutral button or text link, relying on confirmation-dialog friction (see Copywriting Contract) rather than color to signal destructiveness. "Export my data" renders in the default Primary teal (non-destructive action). No red anywhere in Privacy Center, Battery, or Stale-Location UI this phase — full stop.

### 3. Low-battery alert surface (NOTIF-01) — full Notifications Center not required yet

The mockup's Notifications Center (05-1) is a multi-filter feed (All/SOS/Zones/Battery) spanning alert types that don't exist until later phases (SOS = Phase 3, Zones = Phase 4). Building the full filtered feed now would mean building empty filter states for features that don't exist.

**Resolution:** Ship a minimal **in-app banner/snackbar** for the low-battery alert (amber, Caution styling, dismissible) plus a lightweight **Battery alert list item** (left-border amber, `Icons.battery_alert`, plain-language body, timestamp) — reusing exactly the mockup's item visual language — inside a simple ungrouped list (no filter chips needed yet, since "Battery" is the only alert type that exists this phase). Add the All/SOS/Zones filter chips only once those alert types exist (Phase 3/4 concern).

## Spacing Scale

Reuses the Phase 1 locked scale verbatim (`mobile/lib/core/theme/app_spacing.dart`) — do not invent a new scale. Base unit 4px.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps (e.g. battery icon to "72%") |
| sm | 8px | Compact spacing (toggle row internal gaps, stat-tile icon gap) |
| xs-md | 12px | Card internal row gaps (permission/toggle row icon-to-text gap) |
| md | 16px | Default element spacing, field-to-field gaps |
| lg | 24px | Section padding, screen gutter default |
| xl | 32px | Larger layout gaps (used sparingly — e.g. gap above bottom-sheet stat row) |

**Phase 2 specific notes:**
- Bottom sheet top corners: 28px radius (map bottom sheets — destination/stats), per `SYSTEM_DESIGN (1).md` Radius tokens.
- Map marker minimum tap target: 44×44px hit-area even though the visual pin may render smaller — accessibility floor, no exceptions.
- Toggle switches: 46×28px track (matches Phase 1's existing toggle sizing convention).

## Stale-Location & Accuracy Treatment (LOC-03, extends CONTEXT.md D-04)

CONTEXT.md D-04 locks the mechanism (faded pin + translucent accuracy-radius circle); exact values were left open. Locked here:

| Staleness band | Pin opacity | Badge |
|---|---|---|
| < 2 min since last ping | 1.0 (full) | none — live-dot blink motion (opacity 1→0.3, ~1s loop, per `SYSTEM_DESIGN (1).md` Motion tokens) |
| 2–15 min | 0.7 | Caption badge: "Last seen {n} min ago" |
| 15 min–1 hr | 0.45 | Caution-amber badge: "Last seen {n} min ago" |
| > 1 hr | 0.3 (floor — never fully invisible) | Caution-amber badge: "Last seen {relative time}" |

**Accuracy-radius circle:** member-color fill at 15% opacity, 1.5px stroke at 40% opacity of the same color, radius mapped 1:1 to the reported GPS accuracy in meters (min rendered radius 24px screen-space so a highly-accurate fix doesn't visually disappear).

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (phase north-star) | "Turn on location sharing" (permission-priming screen, LOC-05 — the gateway action this phase unlocks everything else through) |
| Secondary link (priming screen) | "Not now" (soft decline — value-first framing per D-10, never coercive) |
| Battery-transparency screen CTA | "Got it" (acknowledges, returns to priming/settings flow) |
| Empty state — Live Map, no one else sharing yet | Heading: "No one to show yet" / Body: "Once a family member turns on location sharing, they'll appear here." |
| Empty state — History timeline, no data yet | Heading: "No history yet" / Body: "Once location tracking starts, {name}'s stays and trips will show up here." |
| Empty state — Privacy Center, no one can see you | "No one can see your location right now. Your guardians will still get your location immediately if you ever trigger SOS." (reassurance that SOS bypasses these toggles entirely, per the SOS non-negotiable architecture) |
| Error state — location signal lost | "Location signal lost — showing {name}'s last known location from {time} ago." |
| Error state — OS permission denied | "Location access is off. Turn it on in Settings so your family can see you're safe." + button "Open Settings" |
| Error state — save/network failure (toggle, preset, etc.) | "Couldn't save that setting. Check your connection and try again." (matches Phase 1's established network-error voice) |
| Error state — export failure | "Couldn't prepare your export. Try again in a moment." |
| Low-battery alert (NOTIF-01) | Title: "Low battery" / Body: "{name}'s phone is at {pct}% — location updates may become less frequent." (plain-language, no fear-mongering) |
| Destructive confirmation — Delete my data | Dialog title: "Delete all your location data?" Body: "This permanently removes your live location, history, and stats from SafePath. Your family won't be able to see past activity anymore. This can't be undone." Buttons: "Cancel" (neutral) / "Delete" (Ink, 700 weight — no red, see Scope Resolution #2) |
| Non-destructive negative action — Stop sharing now (early end of a temporary share) | No confirmation dialog — same low-friction convention as Phase 1's "Decline invite" (immediately reversible, re-enable any time) |
| Duration presets (temporary sharing, D-08) | Chip row: "1 hour" / "4 hours" / "8 hours" / "Custom" |
| Auto-stop progress copy | "Sharing for {duration} · {time remaining} left" (deep-teal card, matches 05-3 mockup) |
