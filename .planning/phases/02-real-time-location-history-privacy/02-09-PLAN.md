## Phase Goal

**As a** family member, **I want to** see my family's live and past location on a shared map and control exactly what I share and with whom, **so that** we can keep each other safe without giving up our privacy.

<objective>
Deliver the Privacy Center end-to-end: the per-data-type × per-recipient sharing toggle matrix (PRIV-02, D-07), temporary time-boxed sharing with a duration picker and remaining-time chip (PRIV-03, D-08), data export + irreversible delete behind a confirmation dialog (PRIV-04, D-09), and a retrievable no-data-resale policy screen (PRIV-05). The toggles are a UI reflection of server-authoritative state (P03) — flipping one calls the backend, which actually enforces it.

Purpose: This closes the phase's privacy story on the client. Design discipline is load-bearing here: NO SOS-red-family color anywhere in the Privacy Center — "Delete my data" uses Ink + confirmation-dialog friction, not red (Scope Resolution #2). Export/Delete sit visually subordinate to the toggle matrix (Visual Hierarchy).
Output: `privacy_api` + `privacy_controller`, `privacy_center_screen` (matrix + temporary sharing + export/delete), `privacy_policy_screen`, the `ToggleRow` shared widget, and Privacy-tab wiring.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| app toggle → PATCH sharing-preferences | Client toggle must reflect, not replace, server enforcement |
| app → export/delete | Acts on the caller's own data only (server-scoped) |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-02 | Information Disclosure | Toggle appearing to work but not enforced | high | mitigate | The toggle PATCHes the server, which is the sole enforcement point (P03); the UI reflects returned server state and reverts on failure |
| T-02-01 | Information Disclosure | Export/delete touching another user's data | high | mitigate | Server scopes both to the authenticated caller (P05); the client never supplies a target user id |
| T-02-12 | Repudiation | Accidental irreversible delete | low | mitigate | UI-SPEC confirmation dialog with explicit "This can't be undone" copy (friction, not color) gates the destructive call |
| T-02-17 | Information Disclosure | Exported JSON leaking on-device | low | accept | Export uses the OS share sheet (`share_plus`); handling of the exported file is delegated to the user's chosen target |
</threat_model>
