<objective>
Wire the Live Map header's own identity indicator (the circular avatar/initial in the "Your family, live" bar) to LocationState.selfPosition so it live-updates when the current user changes their profile photo — closing UAT test 72.

Purpose: Plan 02-16 wired the family member markers (LiveMemberMarker) to the ProfileUpdated stream, but the pre-existing header pin in the same file was left as a hardcoded `const MemberMapPin(label: 'You', ...)` that never reads state. A `const` widget instance is structurally immune to rebuilds, and no avatar fields were ever threaded into it. The self user's live profile data already exists in LocationState.selfPosition (kept current by LocationController._applyProfileUpdate) — the header simply never reads it. This is a one call-site omission, not a missing capability.

Output: A non-const header MemberMapPin fed from state?.selfPosition, plus a regression widget test that fails if the header is ever reverted to a hardcoded/const pin.
</objective>
