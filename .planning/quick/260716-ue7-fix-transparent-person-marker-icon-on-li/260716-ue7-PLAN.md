<objective>
The user's own person marker on the live map renders translucent (~30-45% opacity) instead of solid.
Force the self marker to full opacity while leaving family-member staleness fading untouched.

Purpose: The user is testing live on emulator-5554 and reports the person icon "is kinda transparent."
Output: LiveMemberMarker no longer fades the self marker; a regression test locks the behavior.
</objective>

<root_cause>
Confirmed by code + emulator runtime state + screenshot (not a guess):

- `LiveMemberMarker.build()` in `mobile/lib/features/location/presentation/live_map_screen.dart`
  (around lines 248-259) computes
  `opacity = stalenessFor(DateTime.now().toUtc().difference(location.recordedAtUtc)).opacity`
  and wraps the ENTIRE marker (avatar circle + name pill + presence badge) in
  `Opacity(opacity: opacity)`.
- `stalenessFor()` in `mobile/lib/features/location/application/staleness.dart` returns opacity 1.0
  only when the ping is < 2 min old, then drops to 0.7 (2-15 min), 0.45 (15-60 min), 0.3 (> 1 hr).
- `adb dumpsys location` on emulator-5554 shows the fused/GPS position is static and ~33+ min old,
  so the user's own ping `recordedAtUtc` is well past the 15-min threshold. The self marker therefore
  renders at 0.45 or 0.3 opacity — the "kinda transparent" look.
- Visual confirmation (screencap): the on-map "You" marker (avatar, "You" name pill, "ONLINE" badge)
  is clearly faded, while the SAME user renders fully opaque in the top header card and status rail.
  That header pin (`MemberMapPin`) passes no `recordedAt`, so it stays opacity 1 — which is exactly
  why the same person looks solid in the header but transparent on the map.

The staleness fade is an intentional "this person's data is stale" signal for OTHER family members.
It should never apply to the user's own marker — the device always knows where it is. The correct,
scoped fix is: force `isSelf` markers to full opacity; leave family-member markers unchanged.
</root_cause>
