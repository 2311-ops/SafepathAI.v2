# Quick Task 260720-4dq: Fix F-01 — battery percent doesn't refresh independent of GPS movement

## Task

Battery percentage shown on the Live Map pin and member detail sheet never updates for a
stationary device. Root cause: `LocationController` only re-reads
`batteryLevelProvider` inside `_reportPosition()`, which only fires when the GPS
position stream (`positionStreamProvider`, `distanceFilter: 10`) emits a new fix. A
phone sitting still (e.g. charging on a desk) never triggers a new fix, so
`LiveLocation.batteryPercent` goes stale indefinitely.

## Tasks

1. **Add a movement-independent periodic battery refresh to `LocationController`**
   - Files: `mobile/lib/features/location/application/location_controller.dart`
   - Action: Track the last raw `Position` seen from the position stream in a new
     `_lastKnownPosition` field (set inside `_reportPosition`, which already fires on
     every stream event). Start a `Timer.periodic(Duration(minutes: 5), ...)` in the
     same place the other subscriptions (`_positionSubscription`,
     `_lowBatterySubscription`, etc.) are wired up in `_bootstrap()`, once bootstrap
     ownership is confirmed. The timer callback re-invokes the existing
     `_reportPosition(_lastKnownPosition!)` flow (which already re-reads
     `batteryLevelProvider` and calls `hubClient.reportLocation`), so the refreshed
     battery percent flows through the already-wired broadcast path — no new
     backend endpoint or payload shape. Cancel the timer in `_stop()`'s synchronous
     prefix (before any `await`) AND via a dedicated synchronous `ref.onDispose`
     callback, matching the lifecycle discipline of the other subscriptions
     (cancel on disconnect/dispose, null out the reference, no leaks). Do not
     change `distanceFilter` or the position-triggered `_reportPosition` behavior.
   - Verify: `flutter analyze lib/features/location/application/location_controller.dart`
   - Done: Timer starts once connected, cancels on disconnect/dispose, reuses the
     existing report pipeline.

2. **Add regression tests for the periodic refresh**
   - Files: `mobile/test/features/location/location_controller_test.dart`,
     `mobile/pubspec.yaml` (add `fake_async` dev dependency for deterministic
     `Timer.periodic` control), `mobile/test/features/location/location_permission_gate_test.dart`
     (fix pre-existing widget-test harness interaction with the new timer)
   - Action: Add tests verifying (a) advancing a fake clock by the refresh interval
     without a new position event triggers a battery re-read/re-report, (b) no
     refresh fires before any position has been seen, (c) the timer is cancelled on
     disconnect. Fix the `location_permission_gate_test.dart` "granted permission"
     test, which now needs to dispose its `ProviderContainer` inside the test body
     (not just via `addTearDown`) since `flutter_test`'s pending-timer invariant
     check runs before `addTearDown` callbacks fire.
   - Verify: `flutter test test/features/location/location_controller_test.dart test/features/location/location_permission_gate_test.dart`
   - Done: All tests pass; no pending-timer assertion failures.
