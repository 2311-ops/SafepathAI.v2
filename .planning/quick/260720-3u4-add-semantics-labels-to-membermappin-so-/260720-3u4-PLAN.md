<objective>
Add a `Semantics` label to the shared `MemberMapPin` widget so screen-reader
users hear the member's name/label and their online/staleness status, closing
the accessibility gap on the "You" self-summary pin in the Live Map rail card.

Purpose: `LiveMemberMarker` (the family map markers) already exposes correct
Semantics; `MemberMapPin` (used at exactly one call site — the self-summary pin
at `live_map_screen.dart:415`) has none. This makes the identity/status the pin
conveys visually (label + staleness badge) invisible to assistive tech.

Output: A single `Semantics` wrapper on `MemberMapPin.build()`, plus a focused
widget test locking the announced label for the fresh and stale cases.
</objective>
