# Quick Task 260720-3u4: Add Semantics labels to MemberMapPin Summary

**MemberMapPin's build() Column is now wrapped in a `Semantics(container: true, excludeSemantics: true, label: ...)` node announcing `"<label>, <status phrase>"` as one utterance, closing the accessibility gap on the Live Map self-summary pin.**

## Performance

- **Duration:** 12 min
- **Completed:** 2026-07-20
- **Tasks:** 1
- **Files modified:** 2 (1 modified, 1 created)

## Accomplishments
- `MemberMapPin.build()` now computes `statusPhrase = band.badgeText ?? 'current location'` and `semanticsLabel = '${widget.label}, $statusPhrase'`, then wraps the existing Column in `Semantics(container: true, excludeSemantics: true, label: semanticsLabel, child: Column(...))`
- `excludeSemantics: true` collapses the initials `Text` and the visible staleness badge `Text` into the single computed label instead of exposing them as separate semantics nodes
- No `button: true` was set — the pin's only call site (`live_map_screen.dart:415`) sits in a non-tappable summary header and navigates nowhere, so a button role would misrepresent it to assistive tech
- New `member_map_pin_semantics_test.dart` locks both cases: fresh (`'You, current location'`) and stale at 5 minutes (`'Sam, Last seen 5 min ago'`), using `tester.ensureSemantics()` + `find.bySemanticsLabel(...)`
- `LiveMemberMarker` and the 260720-33j pulse-controller/RepaintBoundary logic were left untouched; no constructor changes; no new dependencies

- `mobile/test/shared_widgets/member_map_pin_semantics_test.dart` - New widget test suite asserting the fresh and stale semantics labels resolve via `find.bySemanticsLabel`

## Decisions Made
- Used `band.badgeText ?? 'current location'` directly for the status phrase (matching the plan's spec) rather than introducing a separate `isLiveSelf`-style branch — `badgeText` is already null exactly in the "current location" case (fresh or no `recordedAt`), so no extra conditional was needed
- Did not add `button: true` semantics, since the pin is a status/identity element, not an interactive control, at its only current call site
