<objective>
Render each family member's live battery percentage on the Live Map, sourced from the existing `LiveLocation.batteryPercent` already parsed into location state — independent of the threshold-triggered `low_battery_banner.dart`.

Purpose: The app already receives and stores `batteryPercent` on every SignalR location ping (`LocationUpdateDto` / `MemberLiveLocationDto` → `LiveLocation.batteryPercent`), but it is only ever surfaced by the amber, dismissible low-battery *alert*. Families should be able to read anyone's current battery level at a glance as a neutral, always-available readout.

Output: A small reusable neutral `BatteryIndicator` widget (battery glyph + "N%"), rendered on the family-member map pin (`LiveMemberMarker`) and in the member-detail sheet (`MemberDetailSheet`). Null battery is hidden gracefully. Widget tests confirm the readout appears given known state and disappears when battery is unknown.

Scope guard: Mobile-only. No backend, no DTO, no SignalR, no controller/state changes — `batteryPercent` is already broadcast on every ping and already lives in `LiveLocation` / `LocationState.members`. Do not touch backend code or `location_controller.dart` / `location_models.dart`.
</objective>

  <behavior>
    - Given percent = 72: renders a neutral battery glyph followed by the text "72%".
    - Given percent = null: renders nothing (SizedBox.shrink) — no "0%", no dash, no placeholder box (satisfies the "render null gracefully, no garbage" constraint).
    - Styling uses ONLY neutral tokens: text/icon in AppColors.bodySecondary (or AppColors.ink), sizing via AppSpacing (xs gap between icon and label per UI-SPEC "Icon-to-label gaps (e.g. battery icon to '72%')").   </behavior>
  <action>
    Create a stateless `BatteryIndicator` widget in `mobile/lib/features/location/presentation/battery_indicator.dart`.

    - Constructor takes a single required nullable field: `final int? percent;` (accept the exact type of `LiveLocation.batteryPercent`, which is `int?`).
    - When `percent == null`, return `const SizedBox.shrink()` (the widget self-hides so callers can place it unconditionally). This is the graceful null path — do NOT clamp null to 0 or render an em-dash box.
    - When `percent != null`, render a compact `Row(mainAxisSize: MainAxisSize.min, children: [...])` of a neutral battery glyph (a Material battery icon such as `Icons.battery_full` or `Icons.battery_5_bar`) at a small size (~14px), a `SizedBox(width: AppSpacing.xs)`, and a `Text('$percent%')`.
    - Color the icon and text with `AppColors.bodySecondary` (neutral, distinct from the amber alert). Use `AppTypography.bodySecondary` for the percent text (or `AppTypography.caption` if a mono numeric reads cleaner at small size) — reuse an existing role, do not invent a new typography role or hardcode hex colors.
    - Wrap in `Semantics(label: 'Battery $percent percent')` so screen readers announce the level.
    - Add a `neutral, always-on readout` doc comment clarifying this is intentionally NOT the low-battery alert and must never adopt caution/SOS styling.

    Then create `mobile/test/features/location/battery_indicator_test.dart` following the `wrap()` + `GoogleFonts.config.allowRuntimeFetching = false` pattern from `live_member_marker_test.dart`:
    - Test A: pump `BatteryIndicator(percent: 72)` inside the MaterialApp wrap, assert `find.text('72%')` findsOneWidget.
    - Test B: pump `BatteryIndicator(percent: null)`, assert `find.textContaining('%')` findsNothing (nothing rendered).
  </action>
