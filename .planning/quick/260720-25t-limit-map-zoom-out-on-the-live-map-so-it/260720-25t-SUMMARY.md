# Quick Task 260720-25t: Limit map zoom-out on the Live Map Summary

**Added a documented `minZoom` floor (9 on the Live Map, 5 on the route-history map) to flutter_map's `MapOptions` so pinch and programmatic zoom-out can no longer collapse either map to a whole-world view.**

## Accomplishments
- `live_map_screen.dart`'s `FlutterMap` `MapOptions` now carries `minZoom: 9`, with an inline comment explaining the OSM zoom-scale rationale (regional/metro-area floor), leaving `initialCenter`/`initialZoom`/all else unchanged
- `route_stats_sheet.dart`'s route-history `FlutterMap` `MapOptions` now carries a deliberately more permissive `minZoom: 5`, documented as needed to fit a completed travel route that can span a wide area while still preventing a world-view collapse
- Added a new widget test in `live_map_screen_test.dart` asserting `options.minZoom == 9` on the populated Live Map
- Extended the existing route-history widget test in `route_stats_sheet_test.dart` to assert `options.minZoom == 5`

## Decisions Made
- The route-history map intentionally uses a different, more permissive floor (5) than the Live Map (9), because it bounds a completed travel route that can legitimately span a wide area (e.g. an intercity trip), whereas the Live Map only ever needs to show a co-located family "right now" at a regional scale.
