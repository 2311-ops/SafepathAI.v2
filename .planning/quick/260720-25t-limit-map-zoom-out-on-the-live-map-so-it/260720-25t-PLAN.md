<objective>
Add a zoom-out floor to the Live Map so pinching (or any programmatic zoom) bottoms out at a regional / metro-area view instead of letting the map shrink all the way to a whole-world view. The user's complaint is specifically about the family Live Map (`live_map_screen.dart`): they can currently pinch-zoom-out to a tiny world map and want it to stop at "can see my region."

The fix is a single new `minZoom` argument on the existing `MapOptions` — a constructor floor that flutter_map applies to both pinch gestures and `MapController.move`. No new widget, no new dependency, and no change to `initialZoom`, `maxZoom`, or any other existing map behavior.

Purpose: Removes the disorienting "zoom out to the whole planet" affordance on the family map; keeps the map anchored at a meaningful family-region scale.
Output: `minZoom: 9` on the Live Map and a justified, more-permissive `minZoom: 5` on the route-history map, each documented with a comment, plus widget tests asserting both floors.
</objective>

# Key facts for the executor:
# - flutter_map is ^8.0.0. `MapOptions` accepts an optional `minZoom` (double?).
#   Setting it clamps the lowest zoom reachable by BOTH pinch-to-zoom-out AND
#   `MapController.move(center, zoom)`. A LARGER number = more zoomed in; a
#   SMALLER number = more zoomed out. OSM's zoom scale runs ~0 (whole world) to
#   ~19-20 (building level); ~9-10 is roughly a large metro region, ~5-6 is a
#   country/multi-state extent.
# - Live Map FlutterMap (live_map_screen.dart, ~line 175-187):
#     MapOptions(initialCenter: cameraTarget, initialZoom: 15)
#   No mapController-clamp / cameraConstraint / fitCamera is present — a plain
#   initialCenter + initialZoom. Add ONLY `minZoom:`; leave initialZoom 15 alone.
# - Route-history FlutterMap (route_stats_sheet.dart, ~line 86-90):
#     MapOptions(initialCenter: initial, initialZoom: routePoints.length >= 2 ? 13 : 15)
#   Also a plain initialCenter + initialZoom (no bounds/CameraFit). If — and only
#   if — a CameraFit/bounds constraint were present, it would be out of scope;
#   from the current source there is none, so just add `minZoom:`.
# - DECISION on the route-history map (made here, not left ambiguous): it IS in
#   scope, but gets a DIFFERENT, more-permissive floor than the Live Map. The two
#   maps serve different jobs. The Live Map shows a co-located family "right now,"
#   so a regional floor (9) is right. The route-history map bounds a COMPLETED
#   travel route that can legitimately span a wide area (e.g. an intercity trip),
#   so it needs to zoom out further to fit the whole route — but still must not
#   collapse to a world map. minZoom 5 (~country/multi-state extent) satisfies
#   both: no world-view collapse, yet generous enough for any realistic recorded
#   daily route.
# - Both existing widget tests already pump their screen and assert
#   `find.byType(FlutterMap)`; read the floor via
#   `tester.widget<FlutterMap>(find.byType(FlutterMap)).options.minZoom`.
#   Use `tester.pump(const Duration(milliseconds: 100))` (NOT pumpAndSettle,
#   which hangs on flutter_map's unresolved network tile requests).
