# Quick Task 260720-33j: Gate MemberMapPin's pulse AnimationController on isSelf Summary

**MemberMapPin no longer builds/starts an AnimationController and ticker for non-self pins; the pulse dot's FadeTransition is now wrapped in a RepaintBoundary to isolate its per-tick repaint.**

## Performance

- **Duration:** 8 min
- **Completed:** 2026-07-20
- **Tasks:** 2
- **Files modified:** 2 (1 modified, 1 created)

## Accomplishments
- `_pulseController`/`_pulseOpacity` changed from unconditional `late final` fields to nullable fields constructed only inside an `if (widget.isSelf)` guard in `initState`, so non-self `MemberMapPin` instances create zero `AnimationController`/`Animation` objects and start no ticker
- `dispose()` uses `_pulseController?.dispose()` so a non-self instance (whose controller was never created) no-ops safely instead of risking a throw on an uninitialized `late final`
- `build()`'s pulse block now guards on `if (isLiveSelf && _pulseOpacity != null)` and wraps the `FadeTransition` presence-dot in a `RepaintBoundary`, isolating its per-tick opacity repaint from the surrounding avatar/circle/border subtree
- New focused widget test suite (`member_map_pin_pulse_test.dart`) locks both the self-path (pulse renders, wrapped by a RepaintBoundary) and non-self path (no pulse, safe dispose) behaviors
- `LiveMemberMarker` and all other widgets left untouched; no new dependency added

## Files Created/Modified

- `mobile/test/shared_widgets/member_map_pin_pulse_test.dart` - New test file covering the isSelf:true pulse+RepaintBoundary path and the isSelf:false no-pulse+safe-dispose path

## Decisions Made
- Gated controller construction on `widget.isSelf` only (not `isLiveSelf`), per the plan's explicit guidance — `isSelf` is fixed for the State's lifetime (a new value requires a fresh `initState`), while `isLiveSelf` is a build-time value that can flip over the widget's life without reconstruction; gating on it would risk using a disposed controller in a stale `FadeTransition` during rebuild
