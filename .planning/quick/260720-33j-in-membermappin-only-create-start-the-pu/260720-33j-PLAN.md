<objective>
In `MemberMapPin` (`mobile/lib/shared_widgets/member_map_pin.dart`), stop unconditionally building and starting a pulse `AnimationController` for every instance. Only create/start it when `widget.isSelf` is true — the only render path where the pulse dot is ever shown (the `if (isLiveSelf)` block requires `widget.isSelf`). Additionally, wrap the pulse's `FadeTransition` subtree in a `RepaintBoundary` so its per-tick opacity repaint stays isolated from the surrounding avatar/circle/border.

Purpose: `MemberMapPin` is a shared, reusable widget currently instantiated with `isSelf: true` at exactly one call site (the "You" rail card at `live_map_screen.dart:415`), but it wastes a ticker + animation object on any `isSelf: false` instance where the pulse never renders. Gating on `widget.isSelf` makes the widget correctly cheap/reusable for any future non-self call site, and the `RepaintBoundary` prevents the pulse's animation from repainting the static avatar around it.
Output: Nullable, `isSelf`-gated pulse fields with a guarded `dispose`; a `RepaintBoundary` isolating the pulse; a focused widget test locking both the self and non-self behavior.
</objective>

  <name>Task 1: Gate the pulse AnimationController on widget.isSelf and isolate its repaints with a RepaintBoundary</name>
  <files>mobile/lib/shared_widgets/member_map_pin.dart</files>
  <behavior>
    - isSelf: false instance -> no AnimationController created, no Animation created, no ticker started; no FadeTransition/pulse dot rendered.
    - isSelf: false instance -> dispose() runs without throwing (controller was never created).
    - isSelf: true instance with a live (non-stale) location -> AnimationController is created in initState, repeats, and the pulse FadeTransition dot renders.
    - The pulse FadeTransition is wrapped in a RepaintBoundary; the avatar/circle/border layout is otherwise unchanged.
  </behavior>
  <action>
Change the two pulse fields from non-nullable `late final` to nullable so an instance can legitimately have no pulse. Declare `AnimationController? _pulseController;` and `Animation<double>? _pulseOpacity;` (drop `late final`). Keep the `with SingleTickerProviderStateMixin` mixin exactly as-is — it tolerates a State that never creates a ticker.

In `initState`, wrap the existing controller + animation construction in a single `if (widget.isSelf) { ... }` guard. Inside that guard, keep the current logic unchanged: assign `_pulseController` to an `AnimationController(vsync: this, duration: const Duration(seconds: 1))..repeat(reverse: true)`, then assign `_pulseOpacity` to `Tween<double>(begin: 1, end: 0.3).animate(CurvedAnimation(parent: _pulseController!, curve: Curves.easeInOut))`. Because the assignment happens before the null-check-elided read, reference the just-assigned controller as `_pulseController!` when passing it to CurvedAnimation (it is non-null on that line since it was assigned immediately above, inside the same guard). Do NOT gate on `isLiveSelf` — that value is not known in initState and reactively creating/disposing a controller when staleness flips would risk using a disposed controller in a stale FadeTransition during rebuild. Gate on `widget.isSelf` only.

In `dispose`, change the unconditional `_pulseController.dispose();` to a null-safe `_pulseController?.dispose();` so a non-self instance (whose controller was never created) no-ops instead of throwing on a null / uninitialized `late final`. Keep the `super.dispose()` call.

In `build`, change the pulse guard from `if (isLiveSelf)` to `if (isLiveSelf && _pulseOpacity != null)` so a future refactor that lets isLiveSelf be true without a controller cannot NPE. Because isLiveSelf already requires `widget.isSelf` (and the controller is always created for isSelf instances), this stays functionally identical for the current single call site while being defensively null-safe. Inside that block, wrap the existing `FadeTransition` (the small animated presence-dot subtree — the `FadeTransition` whose child is the 12x12 `Container` dot) in a `RepaintBoundary`, so the tree becomes `Positioned(child: RepaintBoundary(child: FadeTransition(opacity: _pulseOpacity!, child: Container(...))))`. Pass the opacity as the non-null `_pulseOpacity!` (safe inside the guarded block). Wrap ONLY the FadeTransition — not the outer Stack, Column, or the avatar/circle Containers.

Leave everything else in the file unchanged: the `band`/`circleRadius`/`visualSize`/`isLiveSelf` computations, the identity/avatar/initials rendering, the staleness badge block, and the `_hasAvatar`/`_initialsText` helpers.
  </action>
